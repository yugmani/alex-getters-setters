Introduction to JavaScript Property Getters and Setters

Properties and Property Getters and Setters

  • Property getters and setters allow you to change the default behavior when accessing or modifying object properties.

  • Two types of properties in JavaScript:

    1. Data Properties: Used while working with objects.
    2. Accessor Properties: These are simply the methods. These are executed every time working with a property to access or get a value and to set or change a value.
    3. The function that is executed to access some value is getter.
    4. The function that is executed to set or change a value is setter.
    5. These methods are executed automatically, we don't need to call them explicitly.

  • Property Getters

    1. Property getters or methods are used to access (to get value of some property) properties of objects.
    2. This is like a regular object method where we write the 'get' keyword followed by the name of getter method and parentheses.
    3. It is important to make sure that there is a return statement to return something or value in the body of getter method.

  • Property Setters

    1. When you set or change value of some property JavaScript will execute existing property setter, or setter method, for that property.
    2. The keyword 'set' is used, to define setter method.
    3. In addition, we specify at least one parameter in setter method. Specifying is a way to pass that value to a setter so it can be used.
    4. Unlike the getter method, the setter method doesn't have to return anything. Settter is used to set values and no one probably expects it to return anything.

  • Getters and setters as property wrappers

    1. We use getters and setters to restrict changes to property values. For example, we can reject change of a string value if new value is not a string. Or reject the change if the new string is empty as shown in the previous example.
    2. As shown in the previous example, one use case for getter and setter methods is an if ... else statement to dynamically check the new value before someone access them and/or overrides the old value.
    3. Unfortunately, we can't create getter or setter with the same name as existing property. So, you may change the name of those original properties to may be less user-friendly and use user-friendly names for getter and setter methods.
    4. A well known convention in programming to start property name with an underscore( _ ) to mark it as internal meaning no one should use it directly from outside the object. We use this convention with properties for which we want to add getters and setters.
    5. Therefore, first prefix all properties that will have getter and setter methods with underscore.
    6. Second, create property getters and setters with the same names, but now without the underscore prefix.
    7. This will give us properties with friendly names that we have more control over.

  • Creating getters and setters on the go

    1. Object.defineProperty() is a way to add getter and setter methods to an object that already exists.
    2. This method allows you to add new properties to objects or change existing.
    3. Using this we can add or change accessor properties, property getters and setters.

    4. While using defineProperty() we pass in three parameters.

      1. The first argument is the object we want to update.
      2. The second parameter is the property we want to add or change. In the case of property getters and setters, it is the property for which we want to add getter and/or setter method.
      3. The last parameter is for object with descriptors, such as enumerable, configurable, value and so on. In the case of getters and setters, we replace the object with descriptors with an object containing the getter and setter method.
      4. In this case, the syntax for both, getter and setter is almost same as in previous examples. Only the difference is missing of method names.
